形式和内容

吐槽而已

作为一个 APP 开发者，学习技术是为了更好地做业务，能够快速应对变化。
开发者应当为产品服务，而不是一心只想搞技术。

反观当下

先说性能问题：

  1. 初始化耗时
  2. 首屏启动速度
  3. Native 页面加载速度
  4. H5 页面加载速度
  5. 重复判空操作

可以活用很多性能检测工具。

再说内存问题：

  1. low memory
  2. memory leak

再来看扩展性问题：

  1. 插件化
  2. ReactNative

以上都是实实在在存在的问题，这是内容

那么多“内容”没计划，没方案。

管理项目哪那么复杂，看板、scrum、敏捷、sprint 这些都是形式。

找出当前 APP 存在的问题点，逐个立项，排好期，定好 deadline，开搞。
